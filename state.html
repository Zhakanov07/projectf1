Behavioral Design Pattern

State Pattern

Let an F1 race session change its behavior automatically when it moves between Practice, Qualifying, Race, Safety Car, and Finished states.

01

What is the State Pattern?

The State Pattern lets an object change behavior when its internal state changes. Instead of writing one huge if / else block, each state is represented by a separate class with its own rules.

F1 Analogy

A Formula 1 car behaves differently depending on the session state. In practice, it gathers data. In qualifying, it pushes for one fast lap. During a safety car, it must slow down and cannot race at full speed.

03

JavaScript Example

The context stays the same, but each state object changes how actions work.

class RaceSession {
  constructor() {
    this.state = new PracticeState();
  }

  setState(state) {
    this.state = state;
  }

  accelerate() {
    return this.state.accelerate(this);
  }
}

class PracticeState {
  accelerate() {
    return "Collect telemetry at controlled speed.";
  }
}

class RaceState {
  accelerate() {
    return "Use full race power and attack.";
  }
}
