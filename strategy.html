Pit Stop Strategies

How strategy choices decide race outcomes.

1⃣

One-Stop

Minimum time loss. Used on circuits with low tire wear: Monza, Spa.

2⃣

Two-Stop

Balance of speed and durability. Popular on technical circuits: Barcelona, Singapore.

3⃣

Three-Stop

Aggressive strategy for high-wear races: Silverstone, Sakhir.

Undercut

Pit earlier than your rival. Exit on fresh tires and overtake through pace.

Overcut

Stay out longer. Use clean air to attack when exiting the pits.

Rain Strategy

Wet Inter Slick transitions. The timing of the switch is key to victory in changeable conditions.

04

Aerodynamics

How the car's shape determines speed.

Downforce

A modern F1 car generates up to 1500 kg of downforce at 250 km/h. Theoretically could drive on the ceiling.

Ground Effect

Since 2022, the underbody creates a Venturi effect the main source of downforce comes from beneath the car.

DRS

Drag Reduction System. Opening the rear wing increases straight-line speed by 10-15 km/h.

Vortices

Bargeboards, deflectors and vortex generators direct airflow around the car.

Wing Configuration

High downforce (Monaco) vs low drag (Monza). Setup tailored to each circuit.

Wind Tunnel & CFD

Thousands of hours of wind tunnel testing and computer simulation each season.

06

Object Relationships in Code

How F1 domain objects relate to each other using OOP patterns.

Composition

Team ◆ Engine

The Engine cannot exist without its Team. When the Team is destroyed, the Engine goes with it.

class Engine:
    """Tightly coupled to Team lifecycle."""
    def __init__(self, spec: str, horsepower: int):
        self.spec = spec
        self.horsepower = horsepower

    def start(self) -> str:
        return f"{self.spec} engine firing up ({self.horsepower} HP)"


class Team:
    """Owns its Engine via composition."""
    def __init__(self, name: str, engine_spec: str, hp: int):
        self.name = name
        # Engine is CREATED inside Team  composition
        self._engine = Engine(engine_spec, hp)

    def race(self) -> str:
        return f"{self.name}: {self._engine.start()}"

    # When Team is deleted, Engine is deleted too


ferrari = Team("Scuderia Ferrari", "066/7", 1000)
print(ferrari.race())
# Scuderia Ferrari: 066/7 engine firing up (1000 HP)
The Engine is created inside Team.__init__. It has no independent lifecycle — if the team is dissolved, the engine ceases to exist. This is Composition (filled diamond ◆).
Aggregation

Team ◇ Driver

A Driver can exist independently of a Team. They can transfer between teams.

class Driver:
    """Exists independently of any Team."""
    def __init__(self, name: str, number: int):
        self.name = name
        self.number = number

    def __repr__(self):
        return f"#{self.number} {self.name}"


class Team:
    """Holds references to Drivers  aggregation."""
    def __init__(self, name: str):
        self.name = name
        self.drivers: list[Driver] = []

    def sign_driver(self, driver: Driver):
        self.drivers.append(driver)

    def release_driver(self, driver: Driver):
        self.drivers.remove(driver)

    def lineup(self) -> str:
        return f"{self.name}: {', '.join(str(d) for d in self.drivers)}"


verstappen = Driver("Max Verstappen", 1)
red_bull = Team("Red Bull Racing")
red_bull.sign_driver(verstappen)

# Driver survives if Team is dissolved
red_bull.release_driver(verstappen)
# verstappen still exists independently
The Driver object is passed into the Team from outside. Releasing a driver doesn't destroy them. This is Aggregation (empty diamond ◇).
Inheritance

RaceCar → Vehicle

A RaceCar is a Vehicle with specialized behavior and additional capabilities.

from abc import ABC, abstractmethod


class Vehicle(ABC):
    """Base abstraction for all motorized vehicles."""
    def __init__(self, make: str, top_speed: int):
        self.make = make
        self.top_speed = top_speed

    @abstractmethod
    def accelerate(self) -> str: ...

    def __repr__(self):
        return f"{self.make} (top speed: {self.top_speed} km/h)"


class RaceCar(Vehicle):
    """Specialized vehicle with DRS and KERS."""
    def __init__(self, make: str, top_speed: int, drs: bool = True):
        super().__init__(make, top_speed)
        self.drs = drs
        self.kers_charge = 100

    def accelerate(self) -> str:
        boost = " + DRS open!" if self.drs else ""
        return f"{self.make} accelerating to {self.top_speed} km/h{boost}"

    def deploy_kers(self) -> str:
        self.kers_charge -= 20
        return f"KERS deployed! Charge remaining: {self.kers_charge}%"


rb19 = RaceCar("Red Bull RB19", 370, drs=True)
print(rb19.accelerate())
# Red Bull RB19 accelerating to 370 km/h + DRS open!
print(rb19.deploy_kers())
# KERS deployed! Charge remaining: 80%
RaceCar extends the abstract Vehicle class, inheriting make and top_speed while adding drs and deploy_kers. This is Inheritance (IS-A).
Association

Driver — RaceCar

A Driver uses a RaceCar but neither owns nor contains the other.

class RaceCar:
    def __init__(self, model: str):
        self.model = model

    def drive(self, driver_name: str) -> str:
        return f"{driver_name} is driving the {self.model}"


class Driver:
    def __init__(self, name: str):
        self.name = name

    def get_in(self, car: RaceCar) -> str:
        """Association: Driver uses RaceCar."""
        return car.drive(self.name)


sf23 = RaceCar("Ferrari SF-23")
leclerc = Driver("Charles Leclerc")

print(leclerc.get_in(sf23))
# Charles Leclerc is driving the Ferrari SF-23

# Both objects exist independently
# Neither creates nor owns the other
Neither Driver nor RaceCar owns the other. The driver receives the car as a parameter. Both have independent lifecycles. This is Association (plain line).
07

SOLID Principles in F1 Domain

Each letter of SOLID illustrated with Formula 1 objects.

S

Single Responsibility Principle

A class should have one reason to change. The LapTimer only tracks lap times it does not handle telemetry, fuel or tires.

# BAD: one class does everything
class GodRaceManager:
    def record_lap(self): ...
    def calculate_fuel(self): ...
    def adjust_tire_pressure(self): ...
    def send_telemetry(self): ...


# GOOD: each class has one responsibility
class LapTimer:
    """Only tracks lap times."""
    def __init__(self):
        self._laps: list[float] = []

    def record(self, time: float):
        self._laps.append(time)

    def best_lap(self) -> float:
        return min(self._laps) if self._laps else 0.0

    def average(self) -> float:
        return sum(self._laps) / len(self._laps) if self._laps else 0.0


class FuelManager:
    """Only manages fuel calculations."""
    def __init__(self, capacity: float):
        self.capacity = capacity
        self.current = capacity

    def consume(self, amount: float):
        self.current = max(0, self.current - amount)

    def laps_remaining(self, consumption_per_lap: float) -> int:
        return int(self.current / consumption_per_lap)
O

Open/Closed Principle

Open for extension, closed for modification. New tire strategies are added by creating new classes, not by editing existing ones.

from abc import ABC, abstractmethod


class TireStrategy(ABC):
    """Base strategy  closed for modification."""
    @abstractmethod
    def recommend_stop_lap(self, total_laps: int) -> list[int]: ...

    @abstractmethod
    def compound_sequence(self) -> list[str]: ...


class OneStopStrategy(TireStrategy):
    """Extension: one stop around mid-race."""
    def recommend_stop_lap(self, total_laps: int) -> list[int]:
        return [total_laps // 2]

    def compound_sequence(self) -> list[str]:
        return ["Medium", "Hard"]


class TwoStopStrategy(TireStrategy):
    """Extension: two stops splitting race into thirds."""
    def recommend_stop_lap(self, total_laps: int) -> list[int]:
        return [total_laps // 3, 2 * total_laps // 3]

    def compound_sequence(self) -> list[str]:
        return ["Soft", "Medium", "Hard"]


class WetToSlickStrategy(TireStrategy):
    """New strategy added WITHOUT modifying existing code."""
    def recommend_stop_lap(self, total_laps: int) -> list[int]:
        return [8, total_laps // 2]

    def compound_sequence(self) -> list[str]:
        return ["Wet", "Intermediate", "Soft"]
L

Liskov Substitution Principle

Subtypes must be substitutable for their base type. Any Tire subclass must work wherever a Tire is expected.

class Tire:
    """Base tire with grip and degradation."""
    def __init__(self, compound: str, initial_grip: float):
        self.compound = compound
        self.grip = initial_grip

    def degrade(self, laps: int) -> float:
        self.grip -= laps * 0.02
        return max(0, self.grip)

    def is_safe(self) -> bool:
        return self.grip > 0.2


class SoftTire(Tire):
    """Higher grip, faster degradation."""
    def __init__(self):
        super().__init__("Soft", 1.0)

    def degrade(self, laps: int) -> float:
        self.grip -= laps * 0.05  # degrades faster
        return max(0, self.grip)


class HardTire(Tire):
    """Lower grip, slower degradation."""
    def __init__(self):
        super().__init__("Hard", 0.75)

    def degrade(self, laps: int) -> float:
        self.grip -= laps * 0.01  # degrades slower
        return max(0, self.grip)


def simulate_stint(tire: Tire, laps: int):
    """Works with ANY Tire subtype  LSP guaranteed."""
    for lap in range(1, laps + 1):
        grip = tire.degrade(1)
        if not tire.is_safe():
            return f"{tire.compound}: unsafe at lap {lap}"
    return f"{tire.compound}: completed {laps} laps (grip: {tire.grip:.2f})"


# Both subtypes work seamlessly as Tire
print(simulate_stint(SoftTire(), 20))
print(simulate_stint(HardTire(), 20))
I

Interface Segregation Principle

No client should be forced to depend on interfaces it doesn't use. Separate capabilities into focused protocols.

from typing import Protocol


# BAD: fat interface forces all implementors to define everything
# class ICar:
#     def drive(self): ...
#     def repair(self): ...
#     def send_telemetry(self): ...
#     def display_sponsor_logo(self): ...


# GOOD: segregated interfaces
class Driveable(Protocol):
    def accelerate(self) -> str: ...
    def brake(self) -> str: ...


class Repairable(Protocol):
    def diagnose(self) -> list[str]: ...
    def fix(self, issue: str) -> str: ...


class Telemetric(Protocol):
    def send_data(self) -> dict: ...
    def stream_live(self) -> bool: ...


class RaceCar:
    """Implements all three interfaces."""
    def __init__(self, model: str):
        self.model = model
        self.speed = 0

    # Driveable
    def accelerate(self) -> str:
        self.speed += 50
        return f"{self.model} speed: {self.speed} km/h"

    def brake(self) -> str:
        self.speed = max(0, self.speed - 80)
        return f"Braking! Speed: {self.speed} km/h"

    # Repairable
    def diagnose(self) -> list[str]:
        return ["front wing", "tire wear"]

    def fix(self, issue: str) -> str:
        return f"Fixed {issue} on {self.model}"

    # Telemetric
    def send_data(self) -> dict:
        return {"speed": self.speed, "model": self.model}

    def stream_live(self) -> bool:
        return True


class SafetyCar:
    """Only needs Driveable  not forced to implement Repairable/Telemetric."""
    def accelerate(self) -> str:
        return "Safety car leading the field"

    def brake(self) -> str:
        return "Safety car slowing down"
D

Dependency Inversion Principle

High-level modules should depend on abstractions, not concrete implementations. The RaceEngineer depends on a Strategy interface.

from abc import ABC, abstractmethod


class Strategy(ABC):
    """Abstraction that high-level modules depend on."""
    @abstractmethod
    def plan(self, laps_remaining: int, tire_age: int) -> str: ...


class AggressiveStrategy(Strategy):
    def plan(self, laps_remaining: int, tire_age: int) -> str:
        if tire_age > 8:
            return "BOX BOX BOX  switch to Soft"
        return "Push hard, DRS every lap"


class ConservativeStrategy(Strategy):
    def plan(self, laps_remaining: int, tire_age: int) -> str:
        if tire_age > 20:
            return "BOX BOX  switch to Hard"
        return "Manage tires, save fuel"


class RaceEngineer:
    """High-level module depends on Strategy abstraction."""
    def __init__(self, name: str, strategy: Strategy):
        self.name = name
        self._strategy = strategy  # depends on ABC, not concrete class

    def advise(self, laps_remaining: int, tire_age: int) -> str:
        decision = self._strategy.plan(laps_remaining, tire_age)
        return f"Engineer {self.name}: {decision}"

    def change_strategy(self, new_strategy: Strategy):
        """Can swap strategy at runtime without modifying RaceEngineer."""
        self._strategy = new_strategy


# Usage: inject any Strategy implementation
engineer = RaceEngineer("James", AggressiveStrategy())
print(engineer.advise(20, 10))
# Engineer James: BOX BOX BOX  switch to Soft

engineer.change_strategy(ConservativeStrategy())
print(engineer.advise(20, 5))
# Engineer James: Manage tires, save fuel
print(engineer.advise(20, 5)) # Engineer James: Manage tires, save fuel
